# Mutation Analysis Pipeline (`rigi_analysis.mutation`)

This module forms the point-mutation half of the RIGI (Radiation-Induced Genomic Instability) analysis framework. It takes variant calls (VCF files) generated by upstream tools (like `Mutect2`), extracts mutational signatures using `SigProfiler`, annotates the variants against the reference genome, and assigns temporal patterns across the 4-week experimental timeline. 

The output is a set of dose-merged CSV tables that are subsequently consumed by the SV (Structural Variant) pipeline.

---

## Environment Setup

### 1. `rigi-analysis` Environment
The core Python dependencies are managed through a unified Conda environment. Please refer to the **[Main Setup Instructions](../README.md#setup)** for details on creating and activating the `rigi_analysis` environment.

### 2. Third-Party Tools & Data Environment

**Upstream Variant Calling (`Mutect2`)**
*   The input to this pipeline must be somatic VCF files. These are typically generated using **GATK `Mutect2`**. 
*   *Environment*: GATK requires a Java runtime environment. It is executed prior to this Python pipeline to generate the input `vcf_files/`.

**SigProfiler Reference Genomes**
The `sigprofiler.py` script requires the GRCh38 reference genome to be installed within the `SigProfiler` ecosystem.
*   **Online Installation**: `SigProfiler` will automatically attempt to download the genome on first run if an internet connection is available.
*   **Offline Installation (HPC / SciServer)**: 
    1. Pre-download [`GRCh38.tar.gz`](ftp://alexandrovlab-ftp.ucsd.edu/pub/tools/SigProfilerMatrixGenerator/GRCh38.tar.gz) from the AlexandrovLab FTP.
    2. Provide the `--offline-genome` flag to the `sigprofiler.py` script to load it locally.

**GPU Acceleration (Optional)**
`SigProfilerExtractor` can utilize GPU acceleration for faster matrix extraction. If you intend to use this, ensure that you install a CUDA-compatible version of PyTorch into your virtual environment before running the pipeline:
```bash
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu118
```

## Modules Run Description (Pipeline Stages)

The workflow is orchestrated by `rigi_analysis.workflows.MutationPipeline`. Below is each stage in DAG execution order (stages 1–2 run in parallel; all subsequent stages depend on both completing):

### Stage 1a: Annotation Preprocessing (one-time setup)
*   **Module**: `annotation_preprocessing`
*   **Input**: `--build hg38`, `--annotation-dir <path>`
*   **Action**: Downloads refGene data (if not cached) and builds highly efficient interval trees for the human genome.
*   **Output**: Pickled interval trees used for fast genomic lookups in Stage 4.

### Stage 1b: Signature Extraction
*   **Module**: `sigprofiler`
*   **Input**: Directory containing raw VCF files.
*   **Action**: Wraps `SigProfilerMatrixGenerator` and `SigProfilerExtractor`. Parses VCFs and generates sequence-context matrices (`seqinfo.txt`) and mutational signature decompositions.
*   **Output**: SigProfiler output directories containing `*_seqinfo.txt` files.

### Stage 2: Mutation Preprocessing
*   **Module**: `mutation_preprocessing`
*   **Input**: The output directories from Stage 1b (recursively searches for `*_seqinfo.txt`).
*   **Action**: Parses the SigProfiler layout, cleans the data, and serializes mutations by chromosome and mutation type.
*   **Output**: `{chr}_{type}_mutations.pkl` files (e.g., `all_SNV_mutations.pkl`).

### Stage 3: Genomic Annotation (per mutation type)
*   **Module**: `mutation_annotation`
*   **Input**: Processed mutation `.pkl` files (Stage 2) and the interval trees (Stage 1a).
*   **Action**: Maps every point mutation to its corresponding genomic features (exons, introns, intergenic regions, etc.).
*   **Output**: `all_{type}_annotated.pkl`.

### Stage 4: Temporal Pattern Assignment (per mutation type)
*   **Module**: `mutation_pattern_assignment`
*   **Input**: The annotated mutation pickle files (Stage 3).
*   **Action**: Analyzes presence/absence across the time-series (Weeks 0–3) to assign temporal patterns (e.g., `0T00`, `0TT0`).
*   **Output**: `mutation_annotations_dose_<dose>_<type>.csv` for each dose bin.

### Stage 5: Annotation Merging (per mutation type)
*   **Module**: `merge_annotation`
*   **Input**: Annotated pickles (Stage 3) and pattern CSVs (Stage 4).
*   **Action**: Merges temporal pattern assignments back with genomic annotations. Runs concurrently for each mutation type.
*   **Output**: `{type}_dose_{dose}_merged.csv`. **These are the critical handover files consumed by the SV Pipeline.**

### Stages 6–7: Sankey Visualization (per mutation type)
*   **Modules**: `compute_sankey`, `render_sankey`
*   **Action**: Computes trajectory-aware flows of mutation states across the 4-week timeline and renders Sankey diagrams (Figures 2 and S1).
*   **Output**: JSON trajectory files and `.png` figures.

## End-to-end example

```bash
# One-time setup
rigi-analysis-run annotation_preprocessing --build hg38 --annotation-dir annotations

# Per-VCF run
rigi-analysis-run sigprofiler -i vcf_files -o sigprofiler_output -r GRCh38

rigi-analysis-run mutation_preprocessing \
    --input-dir vcf_files/output/vcf_files \
    --output processed_data \
    --summary summary_data

# Steps 3-5 per mutation type
for mut_type in SNV DBS ID MNS; do
    rigi-analysis-run mutation_annotation \
        -m processed_data/all_${mut_type}_mutations.pkl \
        -a annotations -b hg38 \
        -o annotated_mutations/${mut_type}
    
    # Step 4 expects all_<TYPE>_annotated.pkl naming
    mv annotated_mutations/${mut_type}/annotated_mutations.pkl \
       annotated_mutations/${mut_type}/all_${mut_type}_annotated.pkl
    
    rigi-analysis-run mutation_pattern_assignment \
        -i annotated_mutations/${mut_type} \
        -o pattern_analysis/${mut_type} \
        -m ${mut_type}
    
    rigi-analysis-run merge_annotation \
        --annotated-dir annotated_mutations \
        --pattern-dir pattern_analysis \
        --output-dir merged_analysis \
        --mutation-types ${mut_type} \
        --doses dA dB dC dD dE
done

# Step 6-7: Sankey figures
for mut_type in SNV DBS ID MNS; do
    rigi-analysis-run compute_sankey \
        --input annotated_mutations/${mut_type}/all_${mut_type}_annotated.pkl \
        --output-dir sankey_flows/${mut_type}
    
    rigi-analysis-run render_sankey \
        --trajectories-json sankey_flows/${mut_type}/combined/all_chromosomes_trajectories.json \
        --output sankey_figures/${mut_type}_combined.png \
        --title "Temporal Dynamics for ${mut_type}" \
        --subtitle "All Doses, All Chromosomes"
done
```

## Workflow orchestration

The `MutationPipeline` class in `rigi_analysis.workflows.mutation_workflow` runs all stages above as an async DAG via RADICAL asyncflow. Stages 1a and 1b run in parallel; all other stages are sequenced as documented above. To run the full pipeline programmatically:

```python
import asyncio
from rigi_analysis.workflows import MutationPipeline

config = {...}  # see examples/ for a full config template
asyncio.run(MutationPipeline.run_workflow(config=config))
```

For command-line execution on an HPC cluster, the `rigi-analysis-run` dispatcher invokes each stage individually (see end-to-end example above).
